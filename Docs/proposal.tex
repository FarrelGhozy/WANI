\bab{PENDAHULUAN}

\sub{Latar Belakang}

\textbf{Darurat Digitalisasi UMKM Indonesia}

Usaha Mikro, Kecil, dan Menengah (UMKM) merupakan pilar fundamental perekonomian Indonesia. Data Kementerian Keuangan mencatat bahwa sektor UMKM berkontribusi sekitar \textbf{60\%} terhadap Produk Domestik Bruto (PDB) nasional (setara Rp9.580 triliun) dan menyerap hingga \textbf{97\%} dari total tenaga kerja nasional (sekitar 117 juta orang) \cite{idxchannel2025}. Namun, kesenjangan digital masih menjadi hambatan serius: berdasarkan survei Kementerian Koperasi dan UKM tahun 2022 \cite{kemkop2022survei}, hanya sekitar \textbf{20\%} UMKM yang telah mengadopsi teknologi digital secara aktif.

Kondisi ini diperkuat oleh temuan Badan Riset dan Inovasi Nasional (BRIN) yang menyoroti rendahnya adopsi teknologi digital sebagai faktor utama rendahnya produktivitas per unit UMKM, serta menekankan bahwa kecerdasan buatan (AI) bukan lagi kemewahan melainkan alat bertahan hidup bagi UMKM modern \cite{brin2025}.

Kondisi ini menyebabkan inefisiensi operasional (waktu respons lambat, pesanan terlewat), hilangnya peluang bisnis (pelanggan beralih ke kompetitor), ketidakmampuan analisis data historis untuk pengambilan keputusan, dan ketiadaan \textit{web presence} akibat keterbatasan biaya serta keahlian teknis.

\textbf{WhatsApp: Infrastruktur yang Terabaikan}

WhatsApp telah menjadi infrastruktur komunikasi \textit{de facto} bagi UMKM Indonesia. Dengan \textbf{90,9\%} tingkat penetrasi sebagai platform media sosial paling banyak digunakan \cite{wearesocial2024}. Riset internal Meta menambahkan bahwa \textbf{88\%} masyarakat Indonesia mengirim pesan kepada bisnis setiap minggunya — angka tertinggi secara global \cite{meta2025}. WhatsApp digunakan secara luas untuk menerima pesanan, menjawab pertanyaan, dan membangun hubungan dengan pelanggan. Ironisnya, potensi ini belum dimanfaatkan secara optimal: percakapan bisnis bercampur dengan \textit{chat} pribadi, tidak ada sistem \textit{ticketing} atau pelacakan pesanan, tidak ada integrasi dengan manajemen inventaris, dan bot WhatsApp yang ada di pasaran umumnya tidak memiliki keamanan AI yang memadai.

\textbf{Ancaman Keamanan pada Chatbot AI}

Seiring maraknya adopsi \textit{large language model} (LLM) untuk layanan pelanggan, muncul ancaman serius berupa \textit{prompt injection attack}. OWASP \cite{owasp2025} mengidentifikasikan \textit{prompt injection} (LLM01:2025) sebagai kerentanan nomor satu pada aplikasi LLM. Pelaku UMKM yang menggunakan \textit{chatbot} AI tanpa lapisan keamanan berisiko mengalami kebocoran data sensitif, manipulasi sistem, dan eksfiltrasi konfigurasi internal.

\textbf{Kesenjangan Adopsi AI: Peluang yang Terlewatkan}

Di tengah keterbatasan tersebut, potensi kecerdasan buatan (AI) bagi UMKM sangatlah besar. Laporan Salesforce \cite{salesforce2025} mengungkapkan bahwa \textbf{97\%} pelaku UMKM di Indonesia mengakui implementasi AI dapat meningkatkan pendapatan bisnis mereka, dan 77\% telah menggunakan atau bereksperimen dengan AI. Namun, adopsi AI masih terkendala oleh keterbatasan sumber daya, rendahnya literasi digital, dan kekhawatiran terhadap keamanan data — celah yang tepat dijawab oleh platform terintegrasi seperti WANI.

Dari permasalahan di atas, lahirlah \textbf{WANI} -- sebuah platform omnichannel yang mengintegrasikan WhatsApp Bot bertenaga AI, REST API, dashboard administrasi, dan \textit{website generator} statis, dengan keamanan AI berlapis sebagai standar utama.

\sub{Rumusan Masalah}

Berdasarkan latar belakang yang telah diuraikan, dirumuskan permasalahan sebagai berikut:

\begin{enumerate}
	\item Bagaimana merancang dan membangun sistem \textit{chatbot} WhatsApp bertenaga AI yang mampu memahami dan merespons berbagai intensi pelanggan UMKM secara akurat dan kontekstual?
	\item Bagaimana mengimplementasikan sistem pertahanan keamanan berlapis (\textit{multi-layer defense}) yang mampu melindungi \textit{chatbot} AI dari serangan \textit{prompt injection}, kebocoran data pribadi (PII), dan eksfiltrasi sistem?
	\item Bagaimana mengintegrasikan \textit{chatbot} WhatsApp dengan sistem manajemen produk, pesanan, pelanggan, dan pembayaran dalam satu platform yang kohesif?
	\item Bagaimana menyediakan sistem autentikasi berbasis JWT yang aman untuk melindungi akses dashboard administrasi?
	\item Bagaimana mengimplementasikan \textit{file upload} untuk mendukung unggahan gambar QRIS dan aset toko lainnya?
	\item Bagaimana menyediakan solusi pembuatan \textit{website} statis yang terotomatisasi bagi UMKM tanpa memerlukan keahlian teknis?
	\item Bagaimana memastikan seluruh sistem dapat berjalan dengan biaya operasional yang terjangkau dan data tetap sepenuhnya dimiliki oleh pengguna (\textit{self-hosted})?
\end{enumerate}

\sub{Tujuan dan Manfaat}

\textbf{Tujuan}

\begin{enumerate}
	\item Mengembangkan \textit{chatbot} WhatsApp bertenaga AI dengan \textit{intent detection} multi-kategori yang mampu menangani pesanan, pertanyaan, sapaan, keluhan, dan eskalasi secara otomatis.
	\item Membangun sistem keamanan AI 3-tier (regex $\rightarrow$ ML classifier $\rightarrow$ deep LLM judge) sebagai lapisan pertahanan terhadap serangan \textit{prompt injection}.
	\item Merancang dashboard administrasi terpusat untuk manajemen produk, pesanan, pelanggan, pembayaran, dan konfigurasi sistem.
	\item Mengimplementasikan sistem autentikasi JWT yang aman untuk mengamankan akses dashboard dan API.
	\item Menyediakan \textit{file upload} untuk mendukung unggahan gambar QRIS dan aset toko lainnya.
	\item Mengimplementasikan \textit{website generator} otomatis yang menghasilkan situs web statis UMKM dari data toko dan produk.
	\item Menyediakan solusi \textit{open-source} yang dapat di-\textit{host} sendiri (\textit{self-hosted}) dengan biaya operasional minimal.
\end{enumerate}

\textbf{Manfaat}

\begin{itemize}
	\item \textbf{Bagi UMKM}: Otomatisasi layanan pelanggan 24/7 tanpa perlu menyewa staf \textit{customer service}; peningkatan efisiensi operasional; \textit{web presence} gratis dan otomatis.
	\item \textbf{Bagi Pelanggan}: Respons cepat dan akurat terhadap pertanyaan dan pesanan; kemudahan bertransaksi melalui WhatsApp tanpa perlu aplikasi tambahan.
	\item \textbf{Bagi Pengembang}: Kode sumber terbuka yang dapat dipelajari, dikontribusikan, dan dikembangkan lebih lanjut oleh komunitas.
	\item \textbf{Bagi Akademisi}: Studi kasus implementasi LLM untuk lingkungan nyata dengan keamanan berlapis; referensi pengembangan sistem \textit{chatbot} AI yang aman.
\end{itemize}

\sub{Batasan Aplikasi}

Batasan pengembangan WANI ditetapkan sebagai berikut:

\begin{enumerate}
	\item Platform berfokus pada kanal komunikasi WhatsApp (\textit{multi-device API}) sebagai saluran utama interaksi pelanggan.
	\item Sistem dirancang untuk lingkungan \textit{self-hosted} di atas infrastruktur Linux (Ubuntu/Debian) dengan Docker sebagai opsi deployment.
	\item Model LLM yang digunakan berasal dari penyedia pihak ketiga melalui OpenRouter API (\texttt{deepseek/deepseek-v4-flash:free} sebagai model utama, \texttt{google/gemini-2.0-flash-exp:free} sebagai model cadangan/\textit{fallback}).
	\item Dashboard dikembangkan sebagai \textit{single-page application} (SPA) berbasis web, bukan sebagai aplikasi \textit{mobile native}.
	\item \textit{Website generator} menghasilkan situs statis (HTML/CSS/JS) tanpa \textit{backend} dinamis, cocok untuk dipublikasikan di \textit{platform hosting} statis (GitHub Pages, Netlify, Vercel).
	\item Sistem belum mendukung \textit{multi-tenant} penuh; setiap instance WANI melayani satu bisnis UMKM.
\end{enumerate}

\sub{Model Pengembangan Sistem}

WANI dikembangkan menggunakan metodologi \textbf{Extreme Programming} (XP) dengan pertimbangan sebagai berikut:

\begin{itemize}
	\item Empat sub-proyek independen (\texttt{api/}, \texttt{dashboard/}, \texttt{wa-bot/}, \texttt{web-gen/}) memungkinkan pengembangan paralel dan iteratif, sejalan dengan praktik XP tentang \textit{collective ownership}.
	\item \textit{Evolving requirements} yang khas dalam proyek inovasi membutuhkan respons cepat terhadap perubahan.
	\item Fokus pada kualitas kode dan \textit{testing} yang ketat selaras dengan praktik XP seperti \textit{test-driven development} (TDD) dan \textit{continuous integration}.
	\item Tim kecil (4 orang) ideal untuk \textit{pair programming} dan komunikasi tatap muka langsung.
\end{itemize}

\sub{Tahapan Pengembangan}

Tahapan pengembangan WANI mengikuti siklus XP yang terdiri dari:

\begin{enumerate}
	\item \textbf{Planning} -- Penjabaran \textit{user stories} berdasarkan studi literatur dan observasi terhadap kebutuhan UMKM. Prioritas ditentukan berdasarkan nilai bisnis. Iterasi pengembangan berlangsung 1--2 minggu.
	\item \textbf{Design} -- Prinsip YAGNI (\textit{You Ain't Gonna Need It}) diterapkan secara ketat untuk menghindari kompleksitas yang tidak perlu. Arsitektur modular dengan 16 entitas data utama: USER, STORE, CUSTOMER, CATEGORY, PRODUCT, ORDER, ORDER\_ITEM, PAYMENT, CONVERSATION, MESSAGE, WA\_SESSION, AI\_CONFIG, STORE\_PAYMENT\_METHOD, ACTIVITY\_LOG, USAGE\_COUNTER, WEBSITE.
	\item \textbf{Coding} -- \textit{Continuous integration} dengan pekerjaan paralel per sub-proyek:
	      \begin{itemize}
		      \item \textbf{API}: Arsitektur berlapis \textit{routes} $\rightarrow$ \textit{controllers} $\rightarrow$ \textit{models} $\rightarrow$ Prisma \textit{delegate}.
		      \item \textbf{Dashboard}: \textit{Single-page application} dengan React Router v8 dan state management berbasis hook.
		      \item \textbf{WA Bot}: Baileys 6 dengan autentikasi persisten melalui Prisma + PostgreSQL.
		      \item \textbf{Web-Gen}: Template Astro 7 dengan Tailwind CSS 4.
	      \end{itemize}
	\item \textbf{Testing} -- TDD menggunakan \texttt{bun:test} dengan cakupan: unit test (Zod, utilitas, regex), \textit{integration test} (AI pipeline, circuit breaker, rate limiter), \textit{security test} (3-tier firewall, PII scanner, grounding checker), dan \textit{golden reply test} untuk konsistensi respons.
	\item \textbf{Release} -- Siklus rilis pendek dengan dokumentasi \textit{changelog}. Setiap fitur stabil dirilis secara independen per sub-proyek.
\end{enumerate}

\bab{ANALISIS DESAIN SOFTWARE}

\sub{Analisis Kebutuhan Sistem}

\textbf{Kebutuhan Fungsional}

\begin{enumerate}
	\item Sistem mampu menerima dan merespons pesan WhatsApp secara otomatis menggunakan AI.
	\item Sistem mampu mendeteksi 6 kategori intensi pelanggan: \texttt{order}, \texttt{inquiry}, \texttt{greeting}, \texttt{complaint}, \texttt{unknown}, dan \texttt{escalate}.
	\item Sistem mampu membuat, membaca, memperbarui, dan menghapus data produk, pesanan, pelanggan, kategori, dan metode pembayaran.
	\item Sistem menyediakan autentikasi pengguna berbasis JWT (\textit{register}, \textit{login}, \textit{logout}, \textit{forgot-password}, \textit{reset-password}).
	\item Sistem mampu menangani \textit{file upload} untuk gambar QRIS dan aset toko lainnya.
	\item Sistem mampu menghasilkan \textit{website} statis dari data toko dan produk yang tersimpan, serta mengemasnya dalam arsip ZIP untuk diunduh.
	\item Sistem mampu memfilter dan mendeteksi serangan \textit{prompt injection} dalam 3 lapis keamanan.
	\item Sistem mampu membatasi laju permintaan (\textit{rate limiting}) per pelanggan.
	\item Sistem mampu mendeteksi dan meredaksi data pribadi (PII) sebelum dan sesudah pemrosesan AI.
	\item Sistem menyediakan antarmuka \textit{dashboard} untuk administrasi data bisnis secara \textit{real-time}.
\end{enumerate}

\textbf{Kebutuhan Non-Fungsional}

\begin{itemize}
	\item \textbf{Kinerja}: Latensi respons chatbot $\le$ 5 detik untuk $\ge$ 95\% pesanan normal.
	\item \textbf{Keamanan}: Sistem pertahanan injeksi Tier 1 bekerja dalam $\sim$0ms (sinkron); Tier 2 dalam $\sim$500ms; Tier 3 dalam $\sim$1500ms.
	\item \textbf{Ketersediaan}: WA Bot memiliki \textit{auto-reconnect} dengan interval eksponensial \textit{backoff} jika koneksi terputus.
	\item \textbf{Skalabilitas}: Arsitektur modular memungkinkan setiap sub-proyek di-\textit{scale} secara independen.
	\item \textbf{Portabilitas}: Seluruh komponen berjalan di atas Bun runtime dan dapat di-\textit{containerize} dengan Docker.
\end{itemize}

\sub{Perancangan Kebutuhan Sistem}

\textbf{Arsitektur Sistem}

WANI mengadopsi arsitektur modular tiga-komponen yang saling terhubung melalui protokol HTTP dengan autentikasi Bearer token:

\begin{figure}[H]
	\centering
	\includegraphics[width=\textwidth]{fig-arsitektur.png}
	\caption{Arsitektur Sistem WANI}
	\label{fig:arsitektur}
\end{figure}

Setiap sub-proyek bersifat independen dan dapat dikembangkan, diuji, serta di-\textit{deploy} secara terpisah. \textit{Database} menggunakan PostgreSQL dengan dua basis data terpisah: \texttt{wani\_api} untuk data bisnis dan \texttt{wa\_bot} untuk data sesi WhatsApp.

Autentikasi sistem menggunakan dua mekanisme: \textbf{Bearer API Token} untuk komunikasi mesin-ke-mesin (bot $\rightarrow$ API) dan \textbf{JWT (\textit{JSON Web Token})} untuk autentikasi pengguna dashboard. JWT token memiliki masa berlaku 7 hari dengan \textit{refresh} melalui re-login. \textit{Password} di-\textit{hash} menggunakan bcrypt (\texttt{Bun.password}) sebelum disimpan ke database.

\sub{Fitur dan Alur Kerja Sistem}

\textbf{AI Pipeline (16 Langkah)}

Inti dari kecerdasan WANI terletak pada \textit{AI pipeline} yang memproses setiap pesan melalui 16 langkah terstruktur, dijalankan oleh \textit{step coordinator} dengan pola \textit{break-on-block} (langkah 1--8 dapat menghentikan alur lebih awal jika mendeteksi masalah):

\begin{enumerate}[nosep, leftmargin=*]
	\item \textbf{Normalisasi input} -- NFKC normalization, hapus karakter kontrol/\textit{zero-width}, potong 4000 karakter.
	\item \textbf{Upsert pelanggan \& percakapan} -- Cari/buat Customer dan Conversation aktif.
	\item \textbf{Deduplikasi} -- Skip jika \texttt{waMsgId} sudah diproses.
	\item \textbf{Persistensi inbound} -- Simpan Message dengan role = CUSTOMER.
	\item \textbf{Rate limit} -- \textit{Sliding window} per pelanggan (8 pesan/30 detik + 60/jam).
	\item \textbf{PII scan} -- Deteksi nomor telepon, email, NIK, alamat dalam input.
	\item \textbf{3-tier injection defense} -- Deteksi serangan \textit{prompt injection} dalam tiga lapisan (T1 regex $\rightarrow$ T2 ML classifier $\rightarrow$ T3 deep judge).
	\item \textbf{Budget check} -- Verifikasi batas harian panggilan LLM melalui UsageCounter.
	\item \textbf{Load context} -- Ambil data Store, Products, AiConfig dari database.
	\item \textbf{Build prompt} -- System prompt + 10 pesan terakhir + pesan saat ini dengan delimiter dan \textit{canary token}.
	\item \textbf{LLM completion} -- OpenRouter dengan \textit{circuit breaker}, retry maksimal 2x, \textit{fallback model}.
	\item \textbf{Parsing output} -- Ekstrak JSON, validasi menggunakan Zod \textit{discriminated union} (6 intent).
	\item \textbf{Handle intent} -- Eksekusi aksi sesuai intent: buat Order, eskalasi, dll.
	\item \textbf{Output guardrails} -- Empat sub-lapisan: sanitasi (hapus \textit{code fences}, potong 1500 karakter) $\rightarrow$ deteksi kebocoran (\textit{canary token}, delimiter, system prompt, PII) $\rightarrow$ redaksi PII (ganti dengan placeholder) $\rightarrow$ \textit{grounding check} (verifikasi akurasi faktual untuk intent inquiry/order).
	\item \textbf{Record usage} -- Catat jumlah panggilan LLM dan token ke UsageCounter.
	\item \textbf{Persist outbound} -- Simpan \textit{outbound message}, perbarui status percakapan.
\end{enumerate}

\textbf{Alur Data Utama}

\begin{enumerate}
	\item Pelanggan mengirim pesan via WhatsApp $\rightarrow$ WA Bot menerima pesan.

	\item WA Bot mengirimkan balasan ke pelanggan via WhatsApp.

WANI (\textit{WhatsApp AI Native Integration}) telah berhasil dikembangkan sebagai platform omnichannel yang komprehensif, aman, dan terjangkau bagi pelaku UMKM Indonesia. \subsection*{Lampiran B: Panduan Instalasi}
\addcontentsline{toc}{subsection}{Lampiran B: Panduan Instalasi}

\textbf{Prasyarat Sistem}
\begin{itemize}
	\item \textbf{Bun 1.3+} -- Runtime JavaScript (\url{https://bun.sh})
	\item \textbf{PostgreSQL 17} -- Basis data relasional
	\item \textbf{Git} -- \textit{Version control}
\end{itemize}

\textbf{Langkah Instalasi}

\begin{enumerate}
	\item \textbf{Clone repositori}
	      \begin{lstlisting}[style=bash]
git clone https://github.com/FarrelGhozy/WANI.git
cd WANI
\end{lstlisting}

	\item \textbf{Install dependencies}
	      \begin{lstlisting}[style=bash]
cd api && bun install
cd ../dashboard && bun install
cd ../web-gen && bun install
cd ../wa-bot && bun install
\end{lstlisting}

	\item \textbf{Buat database PostgreSQL}
	      \begin{lstlisting}[style=bash]
createdb -U postgres wani_api
createdb -U postgres wa_bot
\end{lstlisting}

	\item \textbf{Konfigurasi environment variable} \\
	      Salin \texttt{.env.example} menjadi \texttt{.env} di \texttt{api/}, \texttt{wa-bot/}, \texttt{web-gen/}. \\
	      Variabel penting:
	      \begin{itemize}
		      \item \texttt{DATABASE\_*} -- koneksi PostgreSQL
		      \item \texttt{API\_TOKEN} -- \textit{shared secret} autentikasi bot $\leftrightarrow$ API
		      \item \texttt{JWT\_SECRET} -- kunci rahasia untuk menandatangani token JWT
		      \item \texttt{OPENROUTER\_API\_KEY} -- kunci API untuk LLM
	      \end{itemize}

	\item \textbf{Jalankan migrasi database}
	      \begin{lstlisting}[style=bash]
cd api && bunx prisma migrate dev
cd ../wa-bot && bunx prisma migrate dev
\end{lstlisting}

	\item \textbf{Jalankan service} (API harus sebelum Bot)
	      \begin{lstlisting}[style=bash]
# Terminal 1 -- API Server (port 3001)
cd api && bun run src/index.ts

# Terminal 2 -- WhatsApp Bot
cd wa-bot && bun run src/index.ts

# Terminal 3 -- Dashboard (port 5173)
cd dashboard && bun run dev

# Terminal 4 -- Web-Gen (testing)
cd web-gen && bun run generate --slug test \
    --template default
\end{lstlisting}
\end{enumerate}

\textbf{Verifikasi}
\begin{itemize}
	\item Dashboard: \url{http://localhost:5173}
	\item API Server: \url{http://localhost:3001}
	\item Status WhatsApp: \texttt{GET /api/qr/status}
\end{itemize}
